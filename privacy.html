2Principle: Local Data Storage Only

Zementa Weather is designed so that all data is stored exclusively on the user's device. There is no transfer of user data to the developer at any point. The developer has no access — neither read nor write — to any user data. No server-side data collection is technically implemented.

No registration, user account, or login is required to use the app. The app works fully anonymously.

Note on Firebase: Firebase is not actively used by Zementa Weather. No connection to Firebase servers takes place — no authentication, no database usage, no analytics. Inactive code remnants from an earlier development phase may exist but are functionally disabled.

The app processes only the following data, all of which remains on the user's device:

4Purpose of Processing

  • Location data: Single transmission to the Open-Meteo API to retrieve the current weather forecast for the user's location
  • Saved locations: Display weather data for user-selected places within the app
  • App settings: Personalise the app interface according to user preferences
  • Cached weather data: Show last loaded data when offline

Legal basis: Art. 6(1)(a) GDPR (consent) for location data; Art. 6(1)(b) GDPR (performance of contract / provision of app functionality) for all other data.

6Data Sharing

Personal data is never shared with the developer or any third party for advertising, analytics, or commercial purposes. No user profiles are created. No tracking takes place. No analytics or advertising services are used.

The only technically necessary transmission occurs when GPS coordinates or a place name are sent to the weather APIs (Open-Meteo, Nominatim) on the user's initiative.

8Your Rights (GDPR)

As a data subject under the GDPR you have the following rights:

  • Right of access (Art. 15 GDPR)
  • Right to rectification (Art. 16 GDPR)
  • Right to erasure (Art. 17 GDPR) — since all data is stored locally, you can delete it at any time via the app settings or by uninstalling the app
  • Right to restriction of processing (Art. 18 GDPR)
  • Right to data portability (Art. 20 GDPR)
  • Right to object (Art. 21 GDPR)
  • Right to withdraw consent (Art. 7(3) GDPR) — location permission can be revoked at any time in your device settings
Note on data deletion:
Since the developer has no access to your data, you can delete all locally stored data yourself: saved locations via the app settings, all data by uninstalling the app.

For further questions contact qwellcode.codes@gmail.com. You also have the right to lodge a complaint with the competent data protection supervisory authority.

9Children

Zementa Weather is intended for a general audience and is not specifically directed at children under the age of 13. Since no user data is collected or transmitted to the developer, there is no particular risk for minor users. Location data is only used with explicit permission, which for minors should be granted by a parent or guardian.

10Data Security

  • All network communication with external APIs is encrypted via HTTPS/TLS
  • Locally stored data is protected by the Android operating system's app sandbox
  • Location data is discarded immediately after the API request
  • No passwords, authentication tokens, or user accounts are processed — login is neither possible nor required
