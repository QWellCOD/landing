Principle: Local Storage Only

Zementa Weather is designed so that all data is stored exclusively on the user's device. There is no transfer of user data to the developer. The developer has no access to user data at any time.

No account, registration, or login is required to use the app.

Note on Firebase: Firebase is not actively used by Zementa Weather — no connection to Firebase servers, no authentication, no database usage, and no analytics.

Purpose of Data Processing

  • Location data: One-time transmission to the Open-Meteo API for current weather forecast
  • Saved locations: Display of weather data for selected places
  • App settings: Personalisation of the interface
  • Cached data: Offline display

Legal basis: Art. 6(1)(a) GDPR (consent) for location data; Art. 6(1)(b) GDPR for other data.

To exercise your rights: qwellcode.codes@gmail.com

Children and Minors

Zementa Weather does not collect user data. Location data is only used with explicit consent, which should be granted by a parent or guardian for minors.

Data Security

  • All network communication is encrypted via HTTPS/TLS
  • Local data is protected by the Android app sandbox
  • Location data is discarded immediately after the API request
  • No passwords or authentication data are processed
